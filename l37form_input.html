폼 요소와 액션과 입력요소

  • form : 복수의 데이터를 특정 서버에 보내기 위한 양식이다.(회원가입 양식)
  • form.action : 양식을 보내는 리소스 위치(회원가입 처리 액션 페이지)
  • form.name == form.id : 고유 선택자 ==> js forms[name or id] forms.signupForm, forms["signupForm"]
  • input.type=text : 서버에 보낸 데이터를 입력하는 곳
  • input.name : 1. 서버에 보내는 파라미터의 이름, 2. form 요소의 하위 개체(필드)처럼 접근 가능 forms.signupForm.userId
  • input.value : 유저가 입력할 값 또는 기본값
  • 제출 url : http://127.0.0.1:5500/signup.action?userId=WATB&username=%EC%9D%B4%EC%9E%AC%EB%8D%95
  • http://127.0.0.1:5500 : 제출할 리소스가 있는 서버의 위치 (네이버 서버가 실행되고 있는 컴퓨터 위치 == www.naver.com)
  • signup.action : 서버에서 배포하거나 실행중인 리소스 (저장된 파일==path)
  • ?userId=WATB&username=%EC%9D%B4%EC%9E%AC%EB%8D%95 : ? 다음을 쿼리 스트링 파라미터를 규칙대로 나열한 것(userId=WATB&username=이재덕) ,&파라미텉 구분,= : key와 value 구분
  • %EC%9D%B4%EC%9E%AC%EB%8D%95 (이재덕) : 한글을 바이트 단위로 16진수로 표현한것 (url인코딩:한글->16진수로, 공백->+)
  • label : input 요소를 focus 하는 용도로 사용됨 (input 요소가 작거나 완전히 다른 input요소로 만들때)
    • label>input 포함 시켜서 작성 : label에 for을 지워야 한다.
    • label[for=id]을 input#id과 상관없는 외부에 작성 후 label의 for에 id를 대응시킨다.
    • onfocus : input 요소를 작성하기 위해서 클릭하면 발생하는 이벤트
    • onblur : focus를 하고 있다가 벗어나는 상태
  • input [type=checkbox] : 체크박스는 하나의 이름으로 복수의 데이터를 보낼때 사용됩니다.
    • checked : true or false 를 갖는데 존재하면 true 없으면 false 입니다
      • checked="checked"(x) => true
      • checked="true" (x)=> true : 문자열 true는 의미가 없다.
      • checked="false" (x)=> true : 작성하는 것 자체가 true
      • checked (o) 예) forms.form.checkbox.checked=(true or false)
    • onchange : checked의 boolean 데이터가 바뀔때마다 실행되는 이벤트 함수
    • oniput : checkbox를 눌르기만 하면 발생하는 이벤트
    • RadioNodeList : type이 checkbox와 radio인 input을 name 같은 것만 묶어서 배열로 반환
  • input [type=radio] : 라디오 버튼은 하나의 이름으로 복수의 데이터 중 하나를 선택해서 보낼 때 사용됩니다. 나머지는 checkbox와 같은데 선택된 input이 있으면 RadioNodeList의 value가 참조한다.
  • button[type=sumit(생략)]==input[type=sumit] : 양식을 action에 작성된 리소스에 제출하는 버튼
    • onclick, oninput : 누를 때 이벤트가 발생
    • onsubmit : 버튼을 포함하는 form의 submit 이벤트(양식을 제출)를 발생시킨다. (버튼에는 이벤트가 발생하지 않는다.)
    • function(e){} : 이벤ㅌ트를 재정의할 떼 사용할 수 있는 e는 이벤트 정보를 담고 있다. (e.target==this 이벤트가 발생한 대상)
    • e.preventDefult() : 이벤트를 강제 종료하는 함수, 이벤트의 연쇄 작용을 끊는다. 예) 마우스 클릭(onclick(e))-> ( button.input(e), form.submit(e)) (연쇄)
